# Implement Tasks

Execute implementation tasks from a plan, following TDD and constitutional constraints.

Full implementation pending. See design doc Part 3, Phase 2.

## Process

1. **Load plan:** Read the task plan generated by `/adev-plan`.
2. **For each task (in dependency order):**
   a. **Load context:** Read files listed in the task's context routing hints.
   b. **RED phase:** Write a failing test that captures the task's acceptance criteria.
   c. **GREEN phase:** Write the minimum code to make the test pass.
   d. **REFACTOR phase:** Clean up while keeping tests green.
   e. **Constitution gate:** Run quality gate commands from the constitution (test, lint, typecheck). If any fail, fix before proceeding.
   f. **Commit:** Stage and commit with a conventional commit message referencing the task ID.
3. **Specialist routing:** If a task has a specialist hint, load the corresponding prompt from `.context-kit/specialists/` and use it to guide implementation.
4. **Subagent review:** After each task, optionally spawn a review subagent to check the implementation against the spec.

## Arguments

- `--plan <path>`: path to the plan file (required)
- `--task <id>`: execute a single task from the plan
- `--no-commit`: skip automatic commits (useful for exploratory work)
- `--dry-run`: show what would be done without writing code
