# Initialize Context Kit

Scaffold the `.context-kit/` directory and generate initial context files.

## Arguments

- No arguments: greenfield mode (new project)
- `--brownfield`: existing project onboarding

## Greenfield Mode

1. **Scaffold directories:**
   ```
   .context-kit/
   ├── constitution.md
   ├── manifest.yaml
   ├── platform-context.yaml
   ├── specs/
   │   ├── product.md
   │   ├── cross-cutting/
   │   └── features/
   ├── adrs/
   ├── samples/
   ├── orientation/
   │   └── architecture.md
   ├── specialists/
   └── hygiene/              # Add to .gitignore
   ```

2. **Detect tech stack:** Read package.json, tsconfig.json, Cargo.toml, go.mod, requirements.txt, Gemfile, or equivalent. Populate `platform-context.yaml`.

3. **Interactive constitution wizard:** Ask the user about:
   - Project identity (one-line description, repo type)
   - Non-negotiable principles (3-5 core rules)
   - Coding standards (language, conventions)
   - Architecture boundaries (what needs human approval)
   - Quality gate commands (test, lint, typecheck)

   Generate `constitution.md` from answers using the template at `${CLAUDE_PLUGIN_ROOT}/templates/constitution-template.md`.

4. **Generate orientation:** Analyze directory structure, identify key modules, produce a brief `orientation/architecture.md` (3-5 paragraphs describing the codebase layout).

5. **Generate manifest:** Create `manifest.yaml` with detected sync targets (CLAUDE.md always, AGENTS.md if it exists, .cursorrules if .cursor/ exists, copilot-instructions.md if .github/ exists).

6. **Run initial sync:** Execute `/adev-sync` to generate agent files from the constitution.

7. **Update .gitignore:** Add `.context-kit/hygiene/` to .gitignore.

8. **Commit:** Stage and commit `.context-kit/` with message: "feat: initialize adev context kit"

## Brownfield Mode (`--brownfield`)

All greenfield steps, plus:

1. **Absorb existing context:** If CLAUDE.md or AGENTS.md already exists, incorporate their content into the constitution draft (user reviews and approves what to keep).

2. **Reverse-charter:** Analyze top-level directory structure and produce:
   - `specs/product.md` — Product Charter draft from README + detected modules
   - `specs/features/<module>/charter.md` — One Feature Charter draft per detected module

   Mark all drafts clearly: `<!-- DRAFT: Generated by /adev-init --brownfield. Review and refine. -->`

3. **ADR archaeology:** Scan git log for significant changes:
   - Dependency additions/removals (package.json changes)
   - Schema migrations (prisma/migrate, alembic, etc.)
   - Framework upgrades (major version bumps)
   - New service/module creation (new top-level directories)

   Generate retrospective ADR drafts in `.context-kit/adrs/`.

4. **Coverage report:** Generate `.context-kit/hygiene/coverage-report.md` showing which areas have charters and which are uncharted. Prioritize by git change frequency.

## After Initialization

Tell the user:
> "Context Kit initialized at `.context-kit/`. Your constitution has been synced to CLAUDE.md.
> Next steps:
> - Review and refine the constitution: `.context-kit/constitution.md`
> - Review generated charters (if brownfield): `.context-kit/specs/features/`
> - Start specifying work: `/adev-specify`
> - Or brainstorm a new feature: `/adev-brainstorm`"
